# 证据原则

负责证据优先判断、原图优先、补充证据时机，以及金字塔表达原则。

## 证据优先级

- 将事实性主张标记为 `source`、`calculation`、`inference`、`user_judgment` 或 `needs_verification`。
- 不要把弱证据升级成事实。无支撑主张必须标为推断、边界或开放问题。
- 数值、benchmark 胜出、日期、排名、比较和因果主张必须有明确 locator。
- locator 可以是源文本、表格、图、caption、页码、URL、仓库路径或用户陈述。
- 区分 primary source、supplemental research、inference 和 user judgment；不要把它们混进同一种表述口径。

## 原图优先

- 当源图、表格、截图或图示能直接支撑页面主张时，优先使用原始来源图像。
- `ppt_content_brief.md` 中的本地源图路径只能作为 `参考图片` 下的 Markdown 图片引用出现。
- 不围绕图片规定 slide layout。只说明图片展示了什么、支撑了什么主张。
- 如果源图含义模糊、质量低，或只能部分支撑主张，必须在可见表述或内部 notes 中保留边界。
- 最终产物不要 hotlink 远程图片。

## 补充证据时机

只有当已批准观点需要的支撑超出已有 source set 时，才做定向外部研究。

适合补充研究的情况：

- 可见主张依赖可能变化的市场、产品、日期、benchmark 或标准；
- source package 给出了机制，但缺少业务或读者语境；
- 反方证据会实质改变读者判断；
- 用户要求使用当前网页证据。

补充来源必须记录来源、访问日期和作用；不要让补充证据和 primary source 混淆。

## 金字塔表达

- 从读者应该相信的答案开始，再用章节和页面主张支撑它。
- 上层观点概括下层观点。同层观点应属于同一类事物。
- 除非用户明确要求，最多使用三个章节主张。
- 章节标题和页面标题要写成判断，不写成主题标签。
- 将证据翻译成面向读者的判断：为什么重要、为什么可信、边界在哪里。
- 来源支持时，优先使用可量化、可测试的表达。

## 可见观点表达

- `页面标题` 是简短判断钩子，不是主题标签、方法名或抽象类别。
- `标题说明` 承载条件、证据、可测效果、成本或边界。
- `分析总结` 包含 1-3 条可直接使用的 bullet，格式为 `小标题：解释`。
- 只有在确实对应不同决策维度时，才使用多条 `分析总结` bullet。
- 页面角色、证据状态、source locator 和边界说明不要混入可见选项块。

好的页面标题：

- `先复测，再谈上线`
- `可立项，不可直上`
- `不是替代，是补轴`

弱页面标题：

- `R-CLA 路线判断`
- `Depth-wise KV sharing`
- `问题分析`
